Metapolator is a web-based parametric font editor. It provides a GUI for designing with Metafont – a program and language for semi-algorithmic specification of fonts. Metapolator was created out of the need to create large type families efficiently.

A Metafont is produced by specifying the central skeleton of shapes – for which there are "pens" of different angles and widths along the skeleton – rather than the outline of the shapes (with bezier curves for example). As the font shapes are defined through equations, they can be parameterised along axes such as aspect ratio, weight, slant, stroke width, contrast and so on. By defining the design space through parameters, Metafont does not simply design 'fonts' but entire families, hence meta-fonts.

Metapolator can create a Metafont by parsing specially prepared UFO (Universal Font Object) files containing basic font outlines[^1]. The fonts can be meta-designed within the Metapolator interface, for example, a curve needs to always connect to another stem, or the angle of a curve should end from 40° to 60° depending on the width of the glyph. Metapolator allows the designer to utilise Metafont without having to write any Metafont code.

[^1]: Metapolator can parse already existing fonts (such as PostScript Type 1 or OpenType) but they need to be broken up into separate shapes resembling strokes. Technically only on-curve anchor points are required – the off-curve control points are currently not read by Metapolator although these control points can be used by Metafont.
