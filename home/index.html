Introduction

“Metapolator is an open web tool for making many fonts. It supports working in a font design space, instead of one glyph, one face, at a time.

With Metapolator, ‘pro’ font designers are able to create and edit fonts and font families much faster, with inherent consistency. They gain unique exploration possibilities and the tools to quickly adapt typefaces to different media and domains of use.

With Metapolator, typographers gain the possibility to change existing fonts — or even create new ones — to their needs.

Metapolator is extensible through plugins and custom specimens. It contains all the tools and fine control that designers need to finish a font.”
Product Vision by Peter Sikking, UI Architect

Project History

The Metapolator project is a free, libre, open source project that is developed by a community of designers and developers. It began in the March 2013 when Simon Egli (a graphic and type designer from Switzerland) and Dave Crossland (a software freedom activist working to free typography) met in New York City to discuss the Metaflop project that Simon was involved in. Metaflop is web-based graphical user interface to the Metafont parametric font programming language, allowing non-programmers to adjust the parameters of a metafont intuitively.

Simon proposed that the Bezier contours typically drawn by type designers, and stored in the UFO GLIF XML format, could be converted into a Metafont stroke element, if they simply have an even number of points. Dave wondered if this could become a powerful new way to develop typeface families. It would combine the parametric power of Metafont with a graphical interface to glyph drawing, and overcome the inherent limitation of inputting text for outputting graphics that has prevented Metafont from being truly useful for type designers.

Simon returned home to Switzerland and founded the Metapolator project to pursue this dream. He began a PHP codebase and posted some experimental results on Tumblr. Dave and Simon attended the Libre Graphics Meeting 2013 in Madrid in April, and met again in Switzerland to begin a new prototype, mfg. This used a Python web server to connect Robofab, Metapost, mf2pt1, FontForge, and paper.js. Over the following year, the project grew to include new contributors, including Vitaly Volkhov (a Python developer in Ukraine) Wei Huang (a graphic and type designer in Norway) and Nicolas Pauly (a graphic and type designer in Lyon.) With a working prototype in hand, Google Fonts project commissioned Simon to develop a new Sans Serif family, Sean Sans.

In April 2014 Simon presented this prototype at the Libre Graphics Meeting 2014 in Leipzig, which was attended by Lasse Fister (a designer-developer in Nuremburg) and Peter Sikking (a user experience architect in Berlin.) Lasse had developed the ufo.js library, for working with UFO fonts entirely in the browser. He proposed an entirely new approach to Metapolator that would leverage his ufo.js and a native JavaScript implementation of the Hobby Spline to completely move away from Metafont itself. In its place, Lasse proposed Cascading Properties Sheets, which uses the familiar syntax of CSS to describe font properties and parametric formula. At the same time Peter proposed a completely new graphical interface that would be designed from the ground up using his User Experience Architecture process.

The Google Fonts project commissioned font families from Simon, Wei, Nicolas and Lasse, and gave financial support to Peter as well as Reuben Thomas and Ben Martin, software developers who had contributed to FontForge and assisted the software development led by Lasse. These core developers of the “Neue Metapolator” met at the automatic type design conference at the Atelier National de Recherche Typographique in May in Nancy, France.

Development continued over the summer of 2013, posting work in progress and discussions on the Metapolator community on Google Plus. The community grew to include contributions from Coen van der Kamp and Jeroen Breen on the front-end development of the UI that Peter designs. Lasse is developing the core CPS engine, Simon continues to develop the Sean family, and Jeroen is implementing the UI in close collaboration with Peter.

We will present the latest prototype at the Libre Graphics Meeting 2015.

Drawing With Strokes

The core of Metapolator is the stroke element, re-implemented from Metafont. This uses the Hobby Spline to control a stroke, that is defined by points along a skeleton path. Each stroke point has a left point and a right point set at an angle and a distance from that parent point. The parent has a direction angle itself, and a tension value that defines the curvature of the stroke. Here is a simple interactive demonstration to play with drawing the Metapolator way:

Fonts designed for and with Metapolator

Pre-Metapolating
Fonts can be prepared for optimal processing with Metapolator. Each glyph be broken up into its individual strokes, and each stroke should have an equal amount of paired points.
